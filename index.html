總感覺刷Leetcode是資工系的必要任務之一。因此為了完成這必要任務，決定來刷刷Leetcode。截至今天為止，Leetcode上總共有1,007道題，希望半年內可以寫完三分之一的題目。

目前的方式是從Easy開始刷，之後是Medium，最後才是Hard。且只要AC就好，不求Time & Space complexity。當過，AC後還是會在discussion區觀摩大神的想法。今天來記錄兩題還不錯的Easy。

905. Sort Array By Parity

這一題是給定一array，要求先把偶數排前面，奇數再接著放

直覺想法是開一個新array，接著iterate整個given array，遇到偶數將數字放新array，再iterate一次given array，不過這次是遇到奇數才放進新array。

下面是在討論區看到一個滿不錯的解法

建兩個flag,從0以及array尾開始，左邊的找到非偶數，右邊找到非奇數，兩邊皆找到後，互換此兩數，接著再循環下去直到兩邊交會（left >= right）

感覺這有一點點像quicksort的副程式，在pivot左邊的都要比pivot小，右邊的都要比pivot大，若非如此，則兩數交換。

942. DI String Match

這題是給定一個只包含I、D的字串S，回傳任一種排列A，這個排列長度為S長度，裡面的數值從0~S長度，且滿足以下條件：

  • If S[i] == “I”, then A[i] < A[i+1]
  • If S[i] == “D”, then A[i] > A[i+1]

如：

不難看出，如果字串全是D，則排列為從大排到小，全是I，則為從小排到大

個人對這題沒什麼特別想法，純粹由觀察得出這些規律：

  • 第一個I從0開始，之後遞增(1,2,3,…)
  • 第一個D從最大的數（S長度開始），之後遞減(S-1, S-2, S-3, …)

排列A裡面數值的總和，恰好是從0加到S長度，因此可用此性質計算最後一個要放入A的數值（因為A的長度從0開始，比S長度多一）
假設從0加到S長度的總和是total，A內部數值目前是inner，要算最後一個放入A的數值，得先把inner+1（因為I是從0開始）
再用total - inner得到最後要放入A的數值。
